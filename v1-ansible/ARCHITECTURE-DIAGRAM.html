ü§î How Ansible Works (Simple Explanation)

Think of Ansible as a Remote Control üéÆ

1. You press the button (run the script) on your Mac
2. Ansible reads the instructions (playbooks/roles - like a recipe book)
3. Ansible connects over network to Windows server (via WinRM - like SSH for Windows)
4. Ansible runs PowerShell commands remotely on the Windows server
5. Each command does one thing (install AD, reboot, check health, etc.)
6. Ansible reports back whether each step succeeded or failed
7. If something fails, the whole process stops and tells you what went wrong

üí° Key Concept: Agentless

Ansible doesn't need to install any software on the Windows server. It just uses WinRM (Windows Remote Management), which is already built into Windows Server. It's like using Remote Desktop, but for automation instead of manual clicking.

üöÄ How to Run This - Step by Step

First Time Setup (Do Once):

  1. Install Ansible on your Mac:
    brew install ansible pip3 install pywinrm
  2. Navigate to the project folder:
    cd "/Volumes/Vamdev Data/Downloads/Projects/LinkedIn - DC IaaC Build"
  3. Set up encrypted vault for credentials:
    ./scripts/setup-ansible-vault.sh
  4. Add your real credentials to the vault:
    ansible-vault edit inventory/group_vars/all/vault.yml # Add your domain admin username, password, DSRM password
  5. Update inventory with real server names:
    # Edit inventory/production/hosts.yml # Replace example hostnames with your actual DC names

Running the Pipeline:

  1. For Staging/Test Environment:
    ./scripts/run-dc-promotion.sh -e staging -t stg-dc01.staging.linkedin.biz
  2. For Production:
    ./scripts/run-dc-promotion.sh -e production -t lva1-dc03.linkedin.biz # You'll be prompted to type "PROMOTE" to confirm
  3. Dry-Run (Check what would happen without doing it):
    ./scripts/run-dc-promotion.sh -e production -t lva1-dc03.linkedin.biz --check
  4. Health Check Only (No promotion):
    ./scripts/validate-dc-health.sh production lva1-dc03.linkedin.biz
